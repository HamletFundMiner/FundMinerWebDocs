Spending Plan Pending

1. Organization Spending Plan Pending Option

Track funds on spending plan approval status. Spending plans are usually submitted by the fund manager.
refers to a status or flag indicating that a spending plan has not yet been finalized or submitted for a particular fund or set of funds.

Require for specific fund

This option allows you to enforce the requirement of having a spending plan only for specific funds, rather than applying this requirement to all funds or all funds of a certain type.
Let's say your organization has several funds, but only Fund 1 and Fund 4 are part of a critical project that requires strict budgeting and oversight. By using this option, you can ensure that only these two funds (Fund 1 and Fund 4) are flagged as "Spending Plan Pending." This way, these funds cannot be used until a detailed spending plan is submitted and approved, ensuring that they are used according to the organization's financial policies.
It can select multiple founds one by one.

Require for all funds of a given type

If your organization has multiple types of funds, and you only want to enforce the spending plan requirement for "Capital Funds," you would select "Capital Fund" from the dropdown. This means that until a spending plan is in place for each Capital Fund, those funds cannot be used. Other types of funds, such as "Current Use Scholarship," might not require a spending plan if not selected.
Fund Types

Require for all invested funds

Not extra option, all invested founds would need spending plan in order to use the fund.

Require for all funds

No extra, all funds would need Spending Plan.

2. Found Spending Plan Pending Tag

The system administrator has required the submission and approval of a spending plan for this fund. There is no approved spending plan for this fund on file that applies to current fiscal year.

Spending Plan Requirement is the flag field that would trigger the Tag depending on how it was configure in Organization options.

3. Code

if (
fund?.spendingPlanStatus === 'Required' &&
allowedComplianceStatus?.includes('SpendingPlanPending')
) 
{
	let shouldAddSpendingPlanPending = false
		switch (spendingPlanOptions?.target) {
		case 'single':
			if (spendingPlanOptions?.value?.includes(fund.id)) {
				shouldAddSpendingPlanPending = true
			}
		break
		case 'type':
			if (fund.type === spendingPlanOptions?.value) {
				shouldAddSpendingPlanPending = true
			}
		break
		case 'invested':
			if (fund.invested === true) {
				shouldAddSpendingPlanPending = true
			}
		break
		default:
			shouldAddSpendingPlanPending = true
		break
	}
	
	  
	if (shouldAddSpendingPlanPending) {
		statusList.push('SpendingPlanPending')
	}
}

4. Logical condition

  1. If Spending Plan Organization Option is selected AND Fund spending plan required
  2. if Required for specific fund selected AND Selected funds include current fund:
    • Spending Plan Pending Tag Activated
  3. if Require for all funds of a given type selected AND current fund type is one of them:
    • Spending Plan Pending Tag Activated
  4. if Require for all invested funds selected AND current fund type is one of them:
    • Spending Plan Pending Tag Activated
  5. if Require for all funds selected
    1. Spending Plan Pending Tag Activated
